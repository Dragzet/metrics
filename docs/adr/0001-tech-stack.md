# ADR-0001: Выбор технологического стека

## Статус
Принято

## Контекст
Нужно быстро реализовать учебный fullstack CRUD-проект с аутентификацией, ролями, БД и визуализацией.

## Решение
- **Backend**: Go + стандартный `net/http`
- **База данных**: SQLite (`modernc.org/sqlite`) для простого локального запуска
- **Auth**: JWT (`github.com/golang-jwt/jwt/v5`)
- **Пароли**: bcrypt (`golang.org/x/crypto/bcrypt`)
- **Frontend**: HTML/CSS/JS, график через Chart.js
- **Контейнеризация**: Docker

## Альтернативы
- PostgreSQL + ORM: богаче функционально, но выше порог настройки для учебного MVP
- React/Vue SPA: удобнее на больших UI, но больше boilerplate

## Последствия
Плюсы: быстрый старт, мало инфраструктуры, единый репозиторий.
Минусы: SQLite хуже масштабируется при высоком параллелизме.
