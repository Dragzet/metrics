# Презентация проекта: структура слайдов

## Слайд 1: Тема и цель
- Платформа сбора и визуализации данных с датчиков
- Цель: централизовать мониторинг и контроль измерений

## Слайд 2: Анализ предметной области
- Сущности: пользователь, датчик, показание
- Роли: admin, operator, viewer
- Бизнес-сценарии использования

## Слайд 3: Архитектура
- Клиент-серверная модель (3 слоя)
- Компонентная UML-диаграмма (`docs/uml/component.puml`)

## Слайд 4: Аутентификация и авторизация
- JWT flow
- Ролевая модель RBAC
- Диаграмма последовательности (`docs/uml/sequence-auth.puml`)

## Слайд 5: Реализация CRUD
- API endpoints
- Валидация входных данных
- Seed-данные в БД

## Слайд 6: Визуализация данных
- График показаний в веб-клиенте (Chart.js)
- Пример экрана с трендом

## Слайд 7: Тестирование и качество
- Unit/fuzz тесты
- Проверка на некорректные роли и ID в URL

## Слайд 8: DevOps и деплой
- Dockerfile
- Облачный деплой (Render/Fly)

## Слайд 9: Итоги и планы развития
- Что реализовано
- Что можно добавить: алерты, WebSocket/MQTT, отчеты
