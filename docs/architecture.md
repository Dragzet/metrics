# Архитектура клиент-серверного приложения

## Выбранный стиль
Трехуровневая клиент-серверная архитектура:
1. **Presentation Layer**: браузерный клиент (`web/index.html`, `web/app.js`)
2. **Application Layer**: HTTP API (`main.go`)
3. **Data Layer**: SQLite (`db.go`, таблицы `users`, `sensors`, `readings`)

## Компоненты
- **Web Client**: авторизация, вызов CRUD API, визуализация данных через Chart.js
- **API Server**:
  - Аутентификация (`/api/login`, JWT)
  - Авторизация по ролям (admin/operator/viewer)
  - CRUD endpoints для пользователей, датчиков, показаний
- **Database**:
  - хранение пользователей, датчиков, измерений
  - тестовые данные и начальные учетные записи

## Нефункциональные аспекты
- Простота развертывания: один бинарник + встроенный фронтенд
- Безопасность: хеширование паролей bcrypt, токены JWT с истечением срока
- Проверка корректности входных данных: валидация JSON и ролей
